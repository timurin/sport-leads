# ADR-007 — Units of measure directory boundary

**Status:** принято; `v0.8.3` реализован в базовом контуре

## Решение

`UnitOfMeasure` является отдельным persistent-справочником. Он хранит стабильный код, полное название, обозначение, системную категорию величины, точность и признак активности. Базовая единица хранения номенклатуры задаётся nullable-внешним ключом `Nomenclature.storage_unit_id`.

Системные категории: `QUANTITY`, `LENGTH`, `AREA`, `MASS`, `TIME`, `SERVICE`. Первичный набор включает `PCS`, `SET`, `PAIR`, `M`, `M2`, `G`, `KG`, `MIN`, `HOUR`, `SERVICE`.

Legacy-текстовое поле `Nomenclature.unit` сохраняется для обратной совместимости. Миграция создаёт системные единицы, сопоставляет известные legacy-значения и создаёт отдельные legacy-записи для неизвестных непустых значений; данные не удаляются.

В этой итерации не реализуются альтернативные единицы, коэффициенты пересчёта, роли продажи/закупки/списания, упаковки, складской учёт, партии, автоматическая конвертация количества или изменения `SalesOrderItem`.

Новые карточки не могут назначить неактивную единицу. Ранее назначенная единица продолжает отображаться, а системные единицы нельзя деактивировать.
