# ADR-008 — Category custom fields and typed values

**Status:** принято; `v0.8.4` реализован в заявленных границах. **Частично superseded** ADR-015 (`4.8`): typed values и назначения категорий перенесены в `CharacteristicDefinition` / `CategoryCharacteristic` / `NomenclatureCharacteristicValue`; отдельный SoT `CustomField*` снят.

## Решение

Дополнительные реквизиты номенклатуры не добавляются в широкую таблицу `Nomenclature` и не хранятся как произвольный JSON. Источник истины разделён на:

- `CustomFieldDefinition` — стабильный код, название, тип данных, активность, видимость, поиск/фильтрация и необязательная единица измерения;
- `CustomFieldOption` — persistent варианты для `SINGLE_SELECT` и `MULTI_SELECT`;
- `CategoryField` — назначение определения категории с required, inherit, visible, sort order и типизированным default;
- `NomenclatureFieldValue` — typed columns для строк, целых/десятичных чисел, boolean, date и option links.

Поддерживаются `STRING`, `TEXT`, `INTEGER`, `DECIMAL`, `BOOLEAN`, `DATE`, `SINGLE_SELECT`, `MULTI_SELECT` и `COLOR`. Decimal хранится через `Numeric`; `float` не используется.

Итоговая схема категории строится от корня к текущей категории. Дочерняя настройка заменяет родительскую, а `inherit=false` запрещает передачу назначения дальше. Обязательность и defaults проверяются backend-слоем. Индивидуальное значение хранится отдельно и не удаляется автоматически при смене категории; исторические значения остаются доступными для осознанной очистки.

Физическое удаление определения поля не является частью API; используется деактивация. Неактивные поля и варианты не предлагаются для новых назначений, но существующие значения сохраняются и отображаются.

Характеристики, варианты номенклатуры, размеры, SKU, media, модели, производство, склад, цены и НДС остаются за пределами `v0.8.4` и относятся к следующим или отдельным контурам.

Карточка и рабочее место v0.8.5 должны переиспользовать эти API и typed values как единственный источник истины. Допускается переработка UX, дерева категорий и режимов просмотра/редактирования, но не создание второго параллельного редактора или альтернативного хранилища реквизитов.
