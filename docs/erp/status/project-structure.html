ERP-модули

MVP готово

CRM / лиды

Lead, контакты, стадии, создание, конвертация, отказ и история имеют backend-контур.

Готово

Заказы / список и Kanban

Список получает `GET /orders` без demo fallback; перенос карточки сохраняет статус через `PATCH /orders/{id}/status`.

В работе

Заказы / карточка

Read-only карточка читает `GET /orders/{id}`, показывает клиента, ответственного, сумму, исходный лид и backend-историю статусов. Следующий пункт — постоянная организация; пока в settings есть только demo-контур.

Не начато

Производство, склад, закупки

Нет сквозного контура заказ → производство → склад.

Подтверждённые функции

  • Транзакционная конвертация Lead → Client + SalesOrder.
  • Список и карточка заказа используют сохранённые backend-данные.
  • Перенос заказа между Kanban-колонками сохраняет SalesOrder.status и откатывается при ошибке backend.
  • Карточка содержит loading, not found и API error состояния.
  • Денежная сумма заказа хранится через Numeric/Decimal.

Ограничения и ближайший пункт

  • Нет постоянной модели организации и связи заказа с организацией; текущая задача заблокирована до архитектурного решения владельца.
  • Правила базовых переходов и история статусов реализованы; нет изменения заказа, товарных позиций, скидок, НДС, платежей, производства и складских операций.
  • Dashboard заказов остаётся частично demo-контуром.
  • Следующая рекомендуемая итерация: foundation постоянной организации заказа после решения владельца.
