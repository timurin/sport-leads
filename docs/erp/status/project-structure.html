ERP-модули

MVP готово

CRM / лиды

Lead, контакты, стадии, создание, конвертация, отказ и история имеют backend-контур.

Готово

Заказы / список и Kanban

Список получает GET /orders без demo fallback; перенос карточки сохраняет SalesOrder.status через PATCH.

В работе

Заказы / карточка и позиции

Карточка показывает Organization, историю и SalesOrderItem. GET/POST/PATCH/DELETE /orders/{id}/items сохраняют строки и пересчитывают сумму заказа.

Не начато

Производство, склад, закупки

Нет сквозного контура заказ → производство → склад.

Подтверждённые функции

  • Транзакционная конвертация Lead → Client + Organization + SalesOrder.
  • Позиции заказа хранят snapshot-наименование, количество, цену, сумму строки и необязательный material_id.
  • CRUD позиций пересчитывает SalesOrder.amount через Decimal/Numeric и сохраняется после reload.
  • Карточка содержит loading, not found и API error состояния.

Ограничения и ближайший пункт

  • Отдельный Product-каталог, скидки/НДС, оплаты, документы, производство и складские операции не реализованы.
  • Следующая рекомендуемая итерация: ручная проверка v0.7.3 и выбор следующего незавершённого блока заказа.
