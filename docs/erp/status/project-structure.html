ERP-модули

MVP готово

CRM / лиды

Lead, контакты, стадии, создание, конвертация, отказ и история имеют backend-контур.

Готово

Заказы / список и Kanban

Список получает GET /orders без demo fallback; перенос карточки сохраняет SalesOrder.status через PATCH.

В работе

Заказы / карточка

GET /orders/{id} показывает клиента, организацию, ответственного, сумму, исходный лид и историю статусов. Организация сохраняется через PATCH /orders/{id}/organization; следующий пункт — товарные позиции.

Не начато

Производство, склад, закупки

Нет сквозного контура заказ → производство → склад.

Подтверждённые функции

  • Транзакционная конвертация Lead → Client + Organization + SalesOrder.
  • Список и карточка заказа используют сохранённые backend-данные.
  • Перенос заказа между Kanban-колонками сохраняет SalesOrder.status и откатывается при ошибке backend.
  • Карточка содержит loading, not found и API error состояния.
  • Денежная сумма заказа хранится через Numeric/Decimal.

Ограничения и ближайший пункт

  • Организация и связь с заказом постоянны, но settings UI остаётся demo-контуром; исторические заказы допускают nullable organization_id.
  • Реализованы правила переходов и история статусов; нет редактирования заказа, товарных позиций, скидок, НДС, платежей, производства и складских операций.
  • Следующая рекомендуемая итерация: товарные позиции заказа после ручной проверки v0.7.2.
