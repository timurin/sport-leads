В работе

v0.7.x — Заказы покупателей

Готовы SalesOrder, конвертация, backend-список, read-only карточка `/sales/orders/[orderId]`, сохранение статуса Kanban и история статусных переходов.

  • Выполнено: переход из списка, загрузка из backend, loading/not-found/error, ссылка на исходный лид, `PATCH /orders/{id}/status` с откатом UI при ошибке, проверка переходов и `GET /orders/{id}/history` в карточке.
  • Следующий пункт: постоянная организация и связь заказа с ней; сейчас settings содержит только demo-организации, поэтому задача ожидает архитектурного решения.
  • Осталось: товарные позиции, редактирование, оплаты, документы и связь с производством.
  • Критерий завершения этапа: постоянный коммерческий документ с позициями, расчётами и управляемым жизненным циклом.
