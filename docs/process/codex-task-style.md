# Стиль задач Codex

- Задача должна быть компактной, но содержать цель, границы, ограничения, проверки и критерии готовности.
- Одна итерация даёт один законченный результат и не перескакивает через следующий пункт roadmap.
- Не повторять весь roadmap в каждом task-файле; ссылаться на канонические документы и ADR.
- Не создавать commit, tag, push, merge или rebase без разрешения пользователя.
- Не менять код вне явно указанной области задачи.
- Не скрывать ошибки fallback-механизмами.
- Итоговый отчёт кратко перечисляет изменения, файлы, проверки, ограничения и следующую рекомендуемую итерацию.
