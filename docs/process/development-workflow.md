# Процесс разработки

## Роли

- пользователь — владелец продукта и тестировщик;
- ChatGPT — архитектор и партнёр по анализу;
- Codex — разработчик и исполнитель изменений;
- Git — фактическая история изменений;
- канонические документы — память проекта вне истории чатов.

## Цикл

Задача → чтение канонических документов и ADR → анализ кода → реализация → автоматические проверки → визуальная проверка → patch при необходимости → commit/push только с разрешения пользователя → следующая задача.

Перед началом проверяются `AGENTS.md`, `docs/architecture/project-structure.md`, `docs/architecture/erp-check.md`, `docs/roadmap/roadmap.md`, связанные ADR, текущий task, `git status` и последние коммиты.

После итерации обновляются только затронутые канонические документы и оба HTML-статуса. Незавершённые пункты не помечаются `[x]`.
