# Sport Leads v0.5.0 — функциональная контрольная точка раздела «Лиды»

Дата: 17 июля 2026 года.

## Назначение версии

`v0.5.0` фиксирует рабочее состояние раздела «Лиды» перед началом этапа v0.6. Контрольная точка объединяет список и Kanban лидов, detail-страницу лида, локальные рабочие сценарии менеджера, коммуникационный UI и основу коммуникационных коннекторов.

Внешний вид карточки лида в этой версии считается временным и будет финализирован после завершения функционала раздела.

## Реализованные возможности

- список лидов и Kanban с drag-and-drop;
- настраиваемые стадии Kanban с hydration-safe загрузкой из `localStorage`;
- detail-маршрут `/sales/leads/[leadId]` с состояниями загрузки, ошибки и отсутствующего лида;
- шапка лида, данные клиента и контактов, коммерческие параметры;
- синхронизация основного контакта с шапкой лида;
- timeline событий, внутренние заметки и локальные задачи;
- коммуникационная панель с каналами, историей сообщений, вложениями и mock-отправкой;
- адаптивный AppShell и единая модель прокрутки портала;
- общие UI-токены и layout-примитивы дизайн-системы.

## Frontend

Основные части реализации находятся в:

- `frontend/app/(workspace)/sales/leads/[leadId]` — route, loading, error и not-found состояния;
- `frontend/components/sales` — компоненты карточки лида, клиента, коммерческих данных, timeline, заметок, задач и коммуникаций;
- `frontend/lib/sales` — преобразования, валидация и локальные операции домена лида;
- `frontend/lib/demo-data/sales.ts` — демонстрационные данные текущего этапа;
- `frontend/components/layout` и `frontend/components/ui` — каркас и переиспользуемые примитивы;
- `frontend/app/globals.css` — portal-токены и адаптивные правила;
- `frontend/types/sales.ts` — используемые frontend-типы.

Detail-страница построена поверх `AppShell`, `PageContent`, `PageActions` и `ResponsiveGrid`. Основная прокрутка принадлежит `[data-app-shell-main]`; отдельная локальная прокрутка сохранена для списка сообщений.

## Backend communications

Пакет `backend/app/communications` предоставляет независимое от провайдера ядро:

- контракт коммуникационного коннектора;
- registry коннекторов;
- нормализованные команды, сообщения и вложения;
- сервис исходящей отправки и обработки webhook payload;
- классификацию ошибок и правила retry;
- mock-коннектор с проверкой тестовой подписи и in-memory дедупликацией.

Реальные внешние API, endpoint webhook, очередь, repository и миграции сообщений в v0.5.0 не подключены. Текущий communications core не изменяет существующее API приложения.

## Состояние раздела «Лиды»

Demo-лиды `lead-1` и `lead-2` открываются как полноценные рабочие карточки. Неизвестный строковый идентификатор, включая `lead-999`, переводится в локальное состояние «Лид не найден». Числовые идентификаторы могут загружаться через существующий backend API.

Редактирование клиента, контактов и коммерческих параметров выполняется в состоянии текущей страницы. Изменения клиента синхронизируют основной контакт в шапке и добавляют событие в timeline.

## Задачи, заметки и timeline

Задачи поддерживают создание, редактирование, завершение, повторное открытие, перенос срока и удаление. Заметки поддерживают создание, редактирование, удаление, закрепление и упоминания с учётом локальных прав пользователя. Timeline объединяет системные события, комментарии, задачи и коммуникации и предоставляет фильтрацию.

Эти операции пока локальны для открытой страницы и не сохраняются в backend после перезагрузки.

## Коммуникации и коннекторы

Коммуникационная панель поддерживает переключение каналов, отображение входящих и исходящих сообщений, composer и демонстрационные вложения. Отправка создаёт mock-сообщение и событие timeline только в frontend-состоянии.

Backend mock-коннектор проверен отдельно и служит контрактной основой для будущих Telegram, VK, email, WhatsApp и website-адаптеров. Frontend пока не вызывает этот слой через API.

## Дизайн-система

Базовая дизайн-система описана в `docs/design-system` и включает:

- семантические `--portal-*` токены;
- `AppShell`, `AppTopbar`, `PageContent`, `PageActions` и `ResponsiveGrid`;
- общие заголовки, карточки, кнопки, статусы, меню и empty-state компоненты;
- адаптивные правила и checklist миграции страниц.

HTML-шаблон в `docs/example-design` сохранён как использованный визуальный reference, но не является runtime-зависимостью.

## Известные ограничения

- большая часть данных карточки лида является demo-данными;
- клиент, контакты, коммерческие поля, задачи, заметки и сообщения изменяются локально;
- конфигурация стадий Kanban хранится в `localStorage` конкретного браузера;
- реальные коммуникационные API и постоянное хранение сообщений отсутствуют;
- нет очереди отправки, retry worker, Redis и файлового хранилища;
- аутентификация и разграничение прав представлены демонстрационным текущим пользователем;
- внешний вид карточки лида будет финализирован позже;
- автоматический интерактивный browser-smoke при подготовке checkpoint не выполнен: в среде не был доступен browser backend. Маршруты проверены HTTP-smoke, а операции — unit-тестами.

## Следующий этап

В v0.6 планируется продолжить развитие продукта отдельными утверждёнными задачами: связать локальные рабочие сценарии с постоянным хранением и прикладными API, а также развивать интеграции на основе communication connector core. Точный объём v0.6 определяется его task-документами; работы этого этапа в checkpoint не включены.

## Запуск

Backend:

```powershell
docker compose up -d postgres
.\.venv\Scripts\Activate.ps1
cd backend
alembic upgrade head
uvicorn app.main:app --reload
```

Frontend в отдельном терминале:

```powershell
cd frontend
npm ci
npm run dev
```

Основные маршруты:

- `http://localhost:3000/sales/leads`;
- `http://localhost:3000/sales/leads/lead-1`;
- `http://localhost:3000/sales/leads/lead-2`;
- `http://localhost:3000/sales/leads/lead-999` — состояние отсутствующего лида.

- импорт FastAPI и генерация OpenAPI — успешно, дублирующихся маршрутов не обнаружено;
- `npm test` — 22 теста пройдено;
- `npx tsc --noEmit` — успешно;
- `npm run lint` — успешно;
- `npm run build` — успешно, включая динамический route `/sales/leads/[leadId]`;
- HTTP-smoke `/sales/leads`, `/sales/leads/lead-1`, `/sales/leads/lead-2` и `/sales/leads/lead-999` — страницы отрендерены;
- сканирование кандидатного набора — `.env`, ключи, реальные токены и пароли не обнаружены.

Некритичные предупреждения: Starlette сообщает о будущей замене интеграции `httpx`, pytest не смог записать родительский cache-каталог, а Node переводит TypeScript-модули тестов в ESM автоматически. На результаты проверок эти предупреждения не влияют.
