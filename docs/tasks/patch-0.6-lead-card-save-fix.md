Контекст

Во время финальной ручной приёмки patch-0.6-lead-ux-custom-stages обнаружен последний критический дефект.

Все остальные пункты патча приняты:

✅ пользовательские стадии Kanban;
✅ сохранение стадий;

✅ конвертация;
✅ отказ;
✅ drag всей карточки;
✅ график динамики.

Осталась одна проблема.

Проблема

При редактировании карточки лида изменения визуально принимаются, но после сохранения и обновления страницы значения пропадают.

Подтверждено в разделе:

B) Потребность клиента

Вероятнее всего проблема не локальная, а относится ко всему механизму сохранения карточки.

Задача

Не исправлять только одно поле.

Проверить и привести к рабочему состоянию весь механизм сохранения карточки лида.

Необходимо проверить

Frontend

корректную сборку PATCH payload;
отправляются ли новые значения;
не теряются ли поля при сериализации;
соответствие camelCase ↔ snake_case;
обновление локального state после сохранения.

Backend

принимает ли endpoint все поля;
попадают ли поля в schema;
обновляется ли модель;
выполняется ли commit;
корректно ли возвращается обновлённый Lead.
Проверить все редактируемые разделы карточки

Обязательно проверить:

A) Основная информация

B) Потребность клиента

Ответственный

Источник

Особое внимание

Проверить:

PATCH содержит все изменённые поля;
PATCH не отправляет пустой объект;
PATCH не затирает значения undefined;
после успешного сохранения frontend использует актуальные данные backend;
отсутствуют локальные mock-данные;
отсутствуют расхождения между DTO frontend и backend.
Ограничения

Не менять архитектуру.

Не менять дизайн.

Не изменять API без необходимости.

Не создавать новые endpoint.

Использовать существующий механизм сохранения.

Проверки

После исправления должны успешно работать:

изменение существующего значения;
заполнение пустого поля;
очистка значения;
повторное изменение;
reload страницы;
повторное открытие карточки.

Все изменения должны сохраняться в базе.

Критерий готовности

Все редактируемые поля карточки лида сохраняются через backend и после обновления страницы отображают актуальные значения.

После выполнения:

пройти frontend test;
backend test;
TypeScript;

build;

git diff --check.

Commit не создавать.

После завершения предоставить краткий отчёт:

причина проблемы;
что исправлено;
какие файлы изменены;
результаты проверок.
