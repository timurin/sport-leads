# Patch v0.8.8h — код, сохранение и удаление реквизита

## Цель

Исправить механику блока «Реквизиты» без изменения дизайна и схемы карточки номенклатуры.

## Сделать

1. Скрыть поле `Код` при создании реквизита.
2. Код генерировать автоматически на backend/service-уровне.
3. Исправить кнопку `Сохранить`:
   - сохранять typed-значение только текущей номенклатуры;
   - обновлять UI без полной перезагрузки;
   - показывать состояния сохранения и ошибки.
4. Исправить кнопку `Удалить`:
   - удалять только индивидуальное назначение и значение из текущей карточки;
   - не удалять определение из общего справочника;
   - не менять другие карточки;
   - не удалять категорийное назначение.
5. Для унаследованного реквизита:
   - запретить удаление из карточки;
   - разрешить очистку значения только если поле необязательное;
   - показать понятное сообщение пользователю.
6. Удаление общего `CustomFieldDefinition` из карточки запретить.

## Backend

Проверить и исправить существующие API/service:

- сохранение typed-значения;
- удаление индивидуального назначения;
- очистку значения;
- автоматическую генерацию уникального code;
- ошибки 404/409/422.

Не создавать второй механизм custom fields.
