## 1. Исправление DnD

Основной файл:

frontend/components/kanban/kanban-board.tsx

Проверить функцию applyLocalMove и весь поток обработки Drag-and-Drop.

В текущей реализации новое состояние доски сначала вычисляется:

const nextColumns = moveKanbanCard(currentColumns, move);

Затем сохраняется:

columnsRef.current = nextColumns;

После этого то же перемещение повторно вычисляется внутри setColumns:

setColumns((renderedColumns) => moveKanbanCard(renderedColumns, move));

Такое двойное применение одного перемещения во время частых событий dragOver может создавать повторные обновления состояния и приводить к ошибке Maximum update depth exceeded.

### Требуемое направление исправления

Одно перемещение должно рассчитываться только один раз.

Ожидаемая логика:

const currentColumns = columnsRef.current;
const nextColumns = moveKanbanCard(currentColumns, move);

if (nextColumns === currentColumns) {
  return false;
}

columnsRef.current = nextColumns;
lastAppliedMove.current = move;
setColumns(nextColumns);

return true;

Не выполнять повторный вызов moveKanbanCard внутри setColumns.

Не ограничиваться механической заменой одной строки. Изучить и проверить весь поток DnD.

### Обязательно проверить

Проверить функции и состояния:

- applyLocalMove
- handleDragStart
- handleDragOver
- handleDragEnd
- handleDragCancel
- getMove
- lastAppliedMove
- columnsRef
- dragStartColumns
- movedAcrossColumns

Убедиться, что после исправления:

- карточка перемещается между колонками;
- карточка перемещается внутри одной колонки;
- одинаковое перемещение не применяется повторно;
- частые события dragOver не создают цикл обновлений;
- отмена DnD возвращает исходное состояние;
- onMove вызывается не более одного раза после завершения;
- onColumnsChange вызывается не более одного раза после завершения;
- завершённые лиды остаются недоступными для DnD;
- состояние LeadWorkspace не обновляется рекурсивно;
- карточка после отпускания остаётся в правильной колонке.

### Требования к стилям

Для основного введённого текста использовать:

text-slate-900

Для placeholder использовать:

placeholder:text-slate-400

Для обычных полей использовать белый фон:

bg-white

Для поискового поля можно сохранить:

bg-transparent

Но поисковое поле обязательно должно содержать:

text-slate-900 placeholder:text-slate-400

Для select выбранное значение должно отображаться тёмным читаемым цветом.

Для textarea добавить тёмный цвет основного текста и читаемый placeholder.

Существующие размеры, отступы, границы, скругления и focus-стили сохранить.

Ожидаемый общий класс обычного поля:

const fieldClass = "mt-1 h-10 w-full rounded-lg border border-slate-200 bg-white px-3 text-sm text-slate-900 placeholder:text-slate-400 outline-none focus:border-blue-400";

Не менять визуальную структуру формы.
