## 1. PageContent

В файле frontend/components/layout/page-layout.tsx добавь явную ширину контейнера.

Текущий базовый набор классов:

mx-auto min-w-0

Должен стать:

mx-auto w-full min-w-0

Не менять:

- size;
- width;
- className;
- max-width;
- существующие варианты compact, default и spacious.

## 2. PageActions

Для режима:

align="between"

используй адаптивное выравнивание:

justify-start sm:justify-between

На мобильной ширине элементы не должны разъезжаться по краям при переносе строк.

Сохрани:

- flex-wrap;
- min-w-0;
- gap;
- align="start";
- align="end".

Ожидаемая карта выравнивания:

start: justify-start
end: justify-start sm:justify-end
between: justify-start sm:justify-between

## 3. ResponsiveGrid

Проверь CSS-реализацию:

- portal-grid-small;
- portal-grid-medium;
- portal-grid-large.

Grid-классы не должны создавать горизонтальный overflow.

Используй принцип:

grid-template-columns: repeat(auto-fit, minmax(min(100%, MIN_WIDTH), 1fr));

Ориентировочные значения:

- small: 220–240 px;
- medium: 280–320 px;
- large: 360–420 px.

Пример безопасной реализации:

.portal-grid-small {
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 230px), 1fr));
}

.portal-grid-medium {
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 300px), 1fr));
}

.portal-grid-large {
  grid-template-columns: repeat(auto-fit, minmax(min(100%, 390px), 1fr));
}

Не использовать:

grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));

без ограничения min(100%, ...), потому что это может создавать горизонтальную прокрутку на узких экранах.

## 5. AppShell и прокрутка

Сохрани текущую модель AppShell:

- оболочка занимает высоту viewport;
- основной scrollbar находится в элементе с data-app-shell-main;
- sidebar и topbar остаются внутри shell.

Проверь:

- body не получает второй вертикальный scrollbar;
- страницы внутри AppShell не используют собственный h-screen;
- корневые контейнеры страниц не создают второй overflow-y-auto;
- горизонтальный overflow отсутствует;
- локальная прокрутка разрешена только для чата, dialog, dropdown и действительно длинных рабочих списков.

Не менять архитектуру AppShell без необходимости.
