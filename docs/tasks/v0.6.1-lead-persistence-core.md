## Правила целостности

- контакты и коммерческие данные не хранить в JSON;
- у лида максимум один основной контакт;
- смена основного контакта атомарна;
- неизвестный лид или контакт возвращает 404;
- probability и discount percent: 0–100;
- суммы и количества неотрицательные;
- денежные значения хранить через Decimal/Numeric, не float;
- даты и время timezone-aware;
- миграция должна иметь upgrade и downgrade;
- API не возвращает ORM-объекты напрямую;
- бизнес-валидация находится в service-слое;
- существующую конвертацию лида не ломать.

npx tsc --noEmit
npm run lint
npm run build

Ручной сценарий:

1. Открыть lead-1.
2. Изменить ИНН.
3. Сохранить.
4. Выполнить Ctrl+F5.
5. Проверить сохранённый ИНН.
6. Добавить контакт.
7. Повторно открыть страницу.
8. Проверить контакт.
9. Назначить его основным.
10. Изменить коммерческий параметр.
11. Повторно открыть карточку.
12. Проверить сохранение.
13. Открыть lead-999 и получить not-found.
14. Проверить, что конвертация лида продолжает работать.
