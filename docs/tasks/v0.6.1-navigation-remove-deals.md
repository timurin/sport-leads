# v0.6.1 — Упростить навигацию CRM

Контекст

В текущей архитектуре Sport-Lead сущность «Сделка» больше не используется как отдельный объект.

Успешный лид конвертируется напрямую в «Заказ покупателя».

Сейчас в верхнем меню есть выпадающий пункт «Заказы» с подпунктами:
- Все заказы;
- Лиды;
- Сделки.

Пункт «Сделки» ведёт на устаревшую demo-страницу и противоречит текущей архитектуре.

Цель

Упростить верхнюю навигацию и оставить только два самостоятельных пункта:

- Лиды
- Заказы покупателей

Объём работ

1. Найти конфигурацию верхнего меню и связанные компоненты навигации.

2. Убрать выпадающий пункт «Заказы».

3. Добавить две отдельные ссылки верхнего меню:
   - «Лиды» → `/sales/leads`
   - «Заказы покупателей» → `/sales/orders`

4. Полностью убрать пункт «Сделки» из навигации.

5. Удалить ссылки на `/sales/deals` из активной конфигурации интерфейса.

6. Если после изменения останутся неиспользуемые элементы конфигурации или импорты, удалить только их.

7. Не удалять саму страницу `/sales/deals` в рамках этого патча, если она может использоваться тестами или историческим demo-контуром. Удалить только навигационный доступ к ней.

Ограничения

- Не менять дизайн верхнего меню.
- Не менять стили, размеры, отступы, иконки и адаптивное поведение остальных пунктов.
- Не менять страницы лидов и заказов.
- Не менять backend.
- Не добавлять новые маршруты.
- Не затрагивать unrelated-файлы.
- Не выполнять commit, push, merge или tag.

Проверки

- В верхнем меню отображаются отдельные ссылки «Лиды» и «Заказы покупателей».
- Выпадающего меню «Заказы» больше нет.
- Пункт «Сделки» отсутствует.
- «Лиды» открывает `/sales/leads`.
- «Заказы покупателей» открывает `/sales/orders`.
- Активное состояние ссылок работает корректно.
- В активной навигации нет ссылок на `/sales/deals`.
- TypeScript проходит.
- ESLint проходит.
- Production build проходит.
- git diff --check проходит.

Критерии готовности

- Навигация соответствует схеме «Лид → Заказ покупателя».
- Устаревшая сущность «Сделка» больше не видна пользователю.
- Остальная навигация и визуальный стиль не изменены.
- Codex предоставляет краткий отчёт по изменённым файлам и проверкам.
- После отчёта останавливается.
