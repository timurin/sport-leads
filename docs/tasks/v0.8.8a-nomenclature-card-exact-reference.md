# v0.8.8a — Exact Nomenclature Card Reference

## Цель

Перенести утверждённый интерфейс карточки номенклатуры в приложение максимально близко к HTML/CSS-эталону.

## Источник истины

Использовать файл:

`nomenclature-card-reference-v1.html`

Сначала скопировать его в репозиторий, например:

`docs/design/nomenclature-card-reference-v1.html`

HTML содержит точную структуру блоков, сетку, размеры, отступы, карточки, вкладки, таблицы, галерею и responsive breakpoints.

Не интерпретировать дизайн свободно и не создавать альтернативный макет.

## Объём

Перенести в текущую страницу:

`/settings/catalogs/nomenclature/[nomenclatureId]`

следующую структуру:

- шапка карточки;
- кнопка возврата;
- название, артикул, тип, категория и статус;
- действия справа;
- горизонтальные вкладки;
- двухколоночная desktop-сетка;
- блок «Основная информация»;
- блок «Реквизиты»;
- блок «Характеристики и варианты»;
- блок «Изображения и файлы»;
- блок «Карточка»;
- блок «История изменений».

## Обязательное соответствие

Повторить из эталона:

- пропорции колонок;
- размеры карточек;
- радиусы;
- границы;
- тени;
- отступы;
- размеры заголовков;
- подписи над полями;
- расположение кнопок;
- вид таблиц;
- расположение media-галереи;
- поведение вкладок;
- мобильную нижнюю панель действий;
- responsive breakpoints.

Допустимо использовать существующие дизайн-токены проекта, но визуальный результат должен максимально совпадать с эталоном.

## Интеграция существующей логики

Не создавать mock-интерфейс.

Подключить к новой разметке реальные данные:

- системные поля номенклатуры;
- custom fields;
- характеристики;
- значения характеристик;
- варианты;
- изображения;
- главное изображение;
- сортировку media;
- статусы;
- даты;
- текущие Server Actions и API.

Удалить все устаревшие заглушки и надписи:

- «Запланировано в v0.8.6»;
- «Запланировано в v0.8.7».

Функциональность v0.8.6 и v0.8.7 уже реализована и должна отображаться как рабочая.

## Адаптивность

Повторить responsive-поведение эталона:

- desktop: две колонки;
- laptop: две компактные колонки;
- tablet: одна колонка;
- mobile: одна колонка и нижняя панель действий;
- вкладки на узких экранах прокручиваются;
- таблицы имеют локальную прокрутку;
- страница не создаёт общий горизонтальный scroll.

Проверить:

- 1920×1080;
- 1440×900;
- 1366×768;
- 1024×768;
- 768×1024;
- 390×844;
- 360×800.

## Ограничения

На этом шаге:

- не менять backend;
- не менять API-контракты;
- не создавать миграции;
- не переписывать бизнес-логику;
- не добавлять внешние UI-библиотеки;
- не менять другие страницы.

Использовать существующие компоненты, но при необходимости разделить текущий `nomenclature-card.tsx` на небольшие визуальные компоненты.

## Проверки

Выполнить:

- frontend tests;
- TypeScript;
- ESLint;
- production build;
- ручную проверку всех указанных разрешений;
- отсутствие надписей о planned v0.8.6/v0.8.7;
- отсутствие регрессии характеристик и media;
- `git diff --check`.

## Критерии готовности

Задача завершена, когда:

1. Карточка визуально повторяет HTML/CSS-эталон.
2. Все блоки находятся на тех же местах.
3. Все реальные данные подключены.
4. Характеристики отображаются внутри нового интерфейса.
5. Изображения отображаются внутри правой media-панели.
6. Заглушки v0.8.6 и v0.8.7 удалены.
7. Desktop, tablet и mobile соответствуют эталону.
8. Существующая функциональность не сломана.
9. P0/P1 отсутствуют.

В отчёте указать изменённые файлы, визуальные расхождения с эталоном, результаты проверок и `git status`.

Commit и push не выполнять.
