# v0.8.8c — Block editing and async save

## Цель

Исправить редактирование и сохранение карточки номенклатуры.

## Сделать

Каждый блок должен иметь собственную кнопку «Редактировать»:

- Основная информация;
- Реквизиты;
- Характеристики и варианты;
- Изображения и файлы, если применимо.

Редактирование одного блока не должно переводить остальные блоки в режим редактирования.

Для каждого редактируемого блока реализовать:

- Редактировать;
- Сохранить;
- Отмена;
- индикатор несохранённых изменений;
- состояние «Сохранение…»;
- состояние «Сохранено»;
- понятную ошибку сохранения;
- блокировку повторного запроса во время сохранения.

## Основная информация

Проверить сохранение:

- наименования;
- артикула;
- типа;
- категории;
- единицы;
- цены;
- валюты;
- описания;
- активности;
- остальных реально существующих системных полей.

## Реквизиты

Проверить сохранение typed custom fields через существующий API.

Не создавать второй механизм сохранения.

## Характеристики

Использовать существующие API и Server Actions.

Не менять архитектуру характеристик и вариантов.

## Поведение

- после успешного сохранения обновить локальные данные без полной перезагрузки страницы;
- при ошибке введённые значения не терять;
- кнопка «Сохранить» должна реально вызывать backend;
- «Отмена» возвращает последний сохранённый state;
- переход между вкладками не должен молча терять изменения;
- при наличии несохранённых изменений показывать предупреждение.

## Проверки

- regression-тест сохранения каждого блока;
- frontend tests;
- backend tests при необходимости;
- TypeScript;
- ESLint;
- production build;
- git diff --check.

Commit и push не выполнять.
