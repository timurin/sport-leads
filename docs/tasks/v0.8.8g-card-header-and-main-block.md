# v0.8.8g — Card Header and Main Block

## Цель

Привести только шапку страницы и вкладку «Основное» к точному дизайну:

`docs/design/nomenclature-card-reference-v1.html`

Остальные вкладки и их содержимое не менять.

## 1. Шапка страницы

Повторить эталон:

- ссылка «← Назад к номенклатуре»;
- строка с наименованием номенклатуры и зелёным статусом;
- ниже: артикул, тип и полный путь категории;
- справа:
  - кнопка меню `⋮`;
  - «Отмена»;
  - синяя кнопка «Сохранить»;
- сохранить текущую механику редактирования и сохранения;
- убрать лишнее пустое пространство между шапкой и вкладками.

Использовать размеры, отступы, цвета и шрифты напрямую из HTML-эталона.

## 2. Вкладка «Основное»

На desktop повторить двухколоночную композицию эталона:

- левая колонка примерно 65%;
- правая колонка примерно 35%;
- gap как в HTML-эталоне.

### Левая колонка

Только блок «Основная информация».

В режиме просмотра показывать значения в сетке 2 колонки.

В режиме редактирования использовать поля:

- Наименование;
- Наименование для печати;
- Артикул;
- Код;
- Категория;
- Тип номенклатуры;
- Единица измерения;
- Краткое наименование;
- Цена;
- Валюта;
- Описание;
- Активность.

Порядок полей должен совпадать с HTML-эталоном.

В заголовке блока:

- «Основная информация»;
- справа кнопка «Редактировать» либо действия «Отмена / Сохранить».

### Правая колонка

Блок «Карточка»:

- создано;
- изменено;
- статус доступности;
- ID.

Ниже блок «История изменений»:

- создание карточки;
- последнее изменение;
- визуальная timeline как в эталоне.

Не показывать на вкладке «Основное»:

- реквизиты;
- характеристики;
- изображения;
- media uploader.

## 3. Визуальное соответствие

Скопировать из HTML-эталона:

- CSS grid;
- пропорции колонок;
- card padding;
- border;
- radius;
- shadow;
- font sizes;
- label colors;
- input heights;
- button sizes;
- background;
- vertical rhythm.

Не создавать новые значения CSS «на глаз», если они уже есть в эталоне.

## 4. Responsive

Desktop:

- две колонки.

Tablet до 900 px:

- одна колонка;
- блок «Карточка» и история идут после основной информации.

Mobile до 640 px:

- поля по одному;
- верхние кнопки скрываются;
- появляется нижняя панель «Отмена / Сохранить» как в эталоне.

## Ограничения

Не изменять:

- backend;
- API;
- Server Actions;
- сохранение;
- custom fields;
- характеристики;
- варианты;
- media;
- остальные вкладки.

Изменять только JSX и CSS шапки и вкладки «Основное».

## Готовность

- шапка совпадает с эталоном;
- вкладка «Основное» совпадает с эталоном;
- справа отображаются «Карточка» и история;
- нет лишних блоков;
- текущая механика редактирования и сохранения работает;
- другие вкладки не изменены.

Commit и push не выполнять.
