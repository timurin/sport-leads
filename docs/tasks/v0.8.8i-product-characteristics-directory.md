# v0.8.8i — Справочник «Характеристики продукции»

## Критическое ограничение

НЕ ИЗМЕНЯТЬ:

- дизайн карточки номенклатуры;
- схему страницы;
- JSX-композицию;
- CSS;
- размеры блоков;
- вкладки;
- текущую механику реквизитов и media.

Разрешено добавить новый раздел настроек и подключить его к существующему блоку «Характеристики и варианты».

## Цель

Создать в разделе «Настройки» отдельную категорию:

`Характеристики продукции`

В ней вести справочники характеристик, которые затем можно назначать номенклатуре.

Первый обязательный набор:

- Цвет;
- Размер.

## Структура раздела

Добавить маршрут согласно текущим соглашениям, например:

`/settings/catalogs/product-characteristics`

На странице показать список характеристик:

- наименование;
- тип;
- количество значений;
- активность;
- действия редактирования.

Действия:

- создать характеристику;
- открыть;
- редактировать;
- деактивировать.

Физическое удаление используемой характеристики не выполнять.

## Характеристика «Цвет»

Создать системную характеристику:

- Наименование: `Цвет`;
- Код генерируется системой;
- Тип: цвет.

Каждое значение содержит:

- системный ID;
- название цвета;
- HEX-значение;
- активность;
- порядок.

Примеры:

- Чёрный — `#000000`;
- Белый — `#FFFFFF`;
- Красный — `#FF0000`;
- Синий — `#0000FF`.

HEX должен:

- начинаться с `#`;
- содержать 6 шестнадцатеричных символов;
- нормализоваться в верхний регистр;
- валидироваться backend.

В интерфейсе показывать:

- цветной preview;
- название;
- HEX.

## Характеристика «Размер»

Создать системную характеристику:

- Наименование: `Размер`;
- Код генерируется системой;
- Тип: список.

Начальные значения:

- S;
- M;
- L.

Каждое значение содержит:

- системный ID;
- отображаемое значение;
- активность;
- порядок.

## Управление значениями

В карточке характеристики добавить возможность:

- создать значение;
- изменить название;
- изменить HEX для цвета;
- изменить порядок;
- активировать;
- деактивировать;
- удалить неиспользуемое значение.

Если значение уже назначено номенклатуре или варианту:

- не удалять его физически;
- разрешить деактивацию;
- существующие карточки должны продолжать его отображать;
- новое назначение деактивированного значения запретить.

## Подключение к номенклатуре

В существующем блоке «Характеристики и варианты» карточки номенклатуры:

- кнопка «Добавить характеристику» должна загружать активные записи из нового справочника;
- пользователь выбирает `Цвет`, `Размер` или другую созданную характеристику;
- после назначения выбирает допустимые значения;
- использовать существующие связи характеристик и вариантов;
- не создавать второй механизм хранения.

Для цвета отображать:

- название;
- цветной маркер;
- HEX при необходимости.

Для размера отображать компактные значения:

- S;
- M;
- L.

## Повторное использование

Одна характеристика и её значения должны использоваться в нескольких номенклатурах.

Изменение названия или HEX в справочнике:

- обновляет отображение справочных данных;
- не должно изменять сохранённый snapshot старых заказов.

## Backend и API

Использовать существующие модели характеристик v0.8.6, если они покрывают требования.

Расширить их минимально для поддержки:

Не создавать параллельные таблицы при наличии подходящей модели.

Миграцию создавать только при необходимости.

- существуют справочники «Цвет» и «Размер»;
- цвет содержит название и HEX;

- значения можно создавать и редактировать;
